# zh-index-pages

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放与维护一系列独立的 HTML 页面。这些页面可能来源于不同时间、不同主题的静态内容，统一在此进行版本管理与发布。仓库本身不绑定特定域名或网站，仅作为页面归档与展示的容器。

## 目录说明

- `pages/` — 存放所有独立 HTML 页面文件，每个页面以单独文件或子目录形式存在。
- `assets/` — 页面所需的公共资源文件（如 CSS、JavaScript、图片等），按需组织。
- `index.html` — 仓库根目录下的入口页面，可用于展示页面列表或导航。

## 页面归档说明

- 每个页面文件应保持独立，避免相互依赖。
- 页面内引用的资源建议使用相对路径，以便在本地预览或部署时保持兼容。
- 页面标题、内容及样式由原作者自行定义，本仓库仅作存储与分发。

## 维护说明

- 如需添加新页面，请将文件放入 `pages/` 目录，并更新根目录的 `index.html` 索引。
- 请勿在页面中嵌入未经授权的第三方资源或违反相关法规的内容。
- 本仓库不提供自动部署或 CI/CD 配置，使用者可根据需要自行托管至任意静态站点服务。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在浏览器中打开 `index.html` 或 `pages/` 下的任意页面文件。
3. 也可将整个仓库部署至支持静态文件的 Web 服务器或托管平台。
